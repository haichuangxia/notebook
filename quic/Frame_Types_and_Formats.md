一个**packets**可以包括多个**frames**。本节主要是讨乱**QUIC**帧的格式及其语义。几种帧的作用如下表：
| 帧            | 作用                             | 特点   |
| ------------- | -------------------------------- | ------ |
| PADDING frame | 填充，使得数据包达到规定的最小值 | 无内容 |
| PING frame    | 查看对方的连通性                 | 无内容 |

## 1. **PADDING** Frames
没有其他特殊的含义，不过可以用来增加`packet`的大小。如果一个数据包过小，没有达到最小值，或者需要提供对抗流量分析，就需要使用这种帧。
``` C++
PADDING Frame {
Type (i) = 0x00,
}
```

## 2. **PING** Frames
主要的作用就是，看对方是否还在（alive），是否可达（reachability）。**PING**帧的格式为：
``` C ++
PING Frame {
Type (i) = 0x01,
}
```
